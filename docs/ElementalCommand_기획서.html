1. 게임 개요

헥사곤 보드 위의 젬을 드래그하여 시퀀스를 만들고,
캐릭터의 스킬을 발동시켜 적을 처치하는 실시간 퍼즐 배틀 게임.
Fire Water Grass Light Dark 5가지 속성의 전략적 조합과 파티 편성이 핵심.

2. 핵심 게임플레이 루프

스테이지 선택 → 파티 편성(1~4명) → 전투 → 승리/패배 → 스테이지 선택

2.1 전투 흐름

  1. 적이 스폰되고 독립적인 공격 타이머 시작 (실시간)
  2. 플레이어가 헥스 보드에서 인접 젬을 드래그하여 시퀀스 생성
  3. 시퀀스가 현재 캐릭터의 선택된 스킬 요구 젬과 일치하면 스킬 발동 (배율 + 컷인)
  4. 불일치 시 기본 공격 (배율 없음)
  5. 드래그된 젬은 소모 → 즉시 리필 (파티 가중치 기반)
  6. 드래그 완료 시 자동으로 다음 캐릭터로 전환
  7. 적 전멸 = 승리 / 파티 전멸 = 패배

3. 헥사곤 보드 시스템

항목내용
보드 크기5열 x 6행 (헥사곤 그리드)
헥스 반경36px
젬 종류5속성: Fire(F), Water(W), Grass(N), Light(L), Dark(D)
특수 아이템폭탄(B) 4%, 라인클리어(R) 3%
장애물적 공격 시 생성, 상호작용 불가

3.1 젬 스폰 (파티 가중치)

기본 각 속성 20% / 파티 캐릭터 속성 보너스: +8%/캐릭터

부족분은 나머지 속성에 균등 분배

예시: 파티(Fire, Fire, Water, Grass) → Fire 36%, Water 28%, Grass 28%, Light 14%, Dark 14%

3.2 젬 소모 및 리필

  • 드래그 완료 시 경로상의 젬이 소모됨
  • 소모된 셀은 즉시 리필 (파티 가중치 기반 확률)
  • 리필 확률: 폭탄 4% / 라인클리어 3% / 일반 젬 93%
  • 리필 애니메이션: 기존 젬 페이드+확대 → 컬러 버스트 → 새 젬 팝인

3.3 특수 아이템 효과

  • 폭탄(B): 드래그 경로에 포함 시 인접 비장애물 셀도 함께 소모
  • 라인클리어(R): 드래그 경로에 포함 시 해당 행 전체 비장애물 셀 소모
  • 특수 아이템은 스킬/약점 매칭에 포함되지 않음
  • 특수 아이템으로 소모된 속성 젬도 약점 진행에 기여

3.4 장애물

  • 적 공격 성공 시 보드의 일반 젬 1개가 파괴 불가 장애물로 변환
  • 장애물 수 제한 없음
  • 선택/소모/리필 효과 대상 불가

3.5 보드 비주얼

  • 보드 프레임: 반투명 어두운 배경 + 미세한 발광 테두리
  • 드래그 경로: 이중 라인 (넓은 글로우 + 좁은 선명 선)
  • 젬: 판타지 보석 스타일 텍스처 + 선택 시 하이라이트 링 + 스케일업

4. 캐릭터 시스템

4.1 파티 구성

  • 전체 로스터: 5명 중 1~4명 선택
  • 전투 중 드래그 완료마다 자동 순환
  • 사망한 캐릭터는 순환에서 제외

4.3 스킬 시스템

  • 캐릭터당 3개 스킬 보유
  • 전투 중 활성 캐릭터의 스킬 패널에서 스킬 선택 가능
  • 각 스킬: 고유 이름 + 요구 젬 시퀀스(3개) + 데미지 배율
  • 선택된 스킬이 시퀀스 매칭, 컷인 텍스트, 데미지 배율 결정

5.2 적 공격 메커니즘

  • 각 적은 독립적인 공격 타이머 보유
  • 공격 게이지가 1/3 속도로 충전 (느리지만 강력)
  • 공격 성공 시 3배 데미지 + 보드에 장애물 1개 생성
  • 공격 대상: 랜덤 생존 캐릭터

5.3 약점(Weakness) 시스템

  • 각 적은 특정 속성 젬의 색상-수량 조합 요구
  • 순서 무관, 파괴된 속성 젬이 생존한 모든 적의 약점에 누적
  • 약점 UI: 색상별 그룹화 + 진행 카운트 (예: 1/3)
  • 약점 완료 시: 대량 데미지 + 타이머 리셋 + 진행도 초기화

5.4 스테이지 HP 스케일링

  • 적 기본 HP는 데이터상 고정
  • 전투 시 스테이지별 1.75배 지수 성장 적용
  • 공식: 실제HP = 기본HP x 1.75^(stageIndex)

스킬 데미지 = 캐릭터.공격력 x 스킬.배율

적 공격: 랜덤 생존 캐릭터 대상 (3배 데미지) + 보드에 장애물 생성

8. 비주얼 시스템

8.1 아트 방향

모바일 판타지 RPG 스타일. 어두운 마법 전장 배경 위에 화려한 보석 젬과 캐릭터 초상화.

8.2 에셋 전략 (프로시저럴 생성)

  • 배경: 그라데이션 + 악센트 컬러 (메뉴/전투 2종)
  • 젬: 속성별 컬러 보석 텍스처 (Phaser Graphics)
  • 캐릭터 초상화: 속성 컬러 기반 프로시저럴 포트레이트
  • 적 초상화: 적 컬러 기반 프로시저럴 몬스터 아트
  • UI 패널: 카드/슬롯 프레임 텍스처

8.3 에셋 매니페스트

카테고리텍스처 키 패턴예시
젬gem-{type}gem-fire, gem-water
캐릭터portrait-{id}portrait-warrior, portrait-mage
적enemy-{id}enemy-goblin, enemy-orc
배경고정 키bg-menu, bg-battle
UI고정 키ui-stage-card, ui-party-card

8.4 화면별 비주얼

  • 스테이지 선택: 판타지 메뉴 배경 + 스테이지 카드 + 호버 틴트
  • 파티 선택: 캐릭터 카드 + 초상화 + 속성 프레임 + 선택 틴트 + 스탯
  • 전투: 전투 배경 + 적 초상화/HP/게이지/약점 + 캐릭터 초상화/HP/스킬패널 + 보석 젬 보드

13. UX 온보딩 시스템 (v0.1.0 추가)

속성 상성표 HUD 추가

전투 화면 우측 상단에 항상 표시되는 미니 속성 상성 참조 패널. 축약 형태(F>N>W>F, L↔D)로 상시 노출하여 전략적 판단 보조.

파티 선택 화면 속성 뱃지 표시

파티 선택 화면의 각 캐릭터 카드에 속성 색상 뱃지를 추가 표시. 캐릭터 카드 하단에 현재 파티 편성 기준 예상 젬 가중치 미리보기 표시.

전투 중 키 힌트 오버레이

첫 번째 전투 진입 시 자동 노출되는 조작 힌트 오버레이. battleHintSeen = true 저장 후 재노출 없음.
