**작성일:** 2026-05-14
**프로젝트:** 27_ElementalCommand
**플랫폼:** 웹 (Phaser.js) → 모바일 (Capacitor) → Steam (Electron)

---

## 1. 개요

젬을 드래그하여 캐릭터 스킬 시퀀스를 완성하면 아군이 적을 공격하는 퍼즐 배틀 게임.
적은 실시간으로 공격해오며, 플레이어는 빠르게 시퀀스를 완성해 아군을 순환시키며 싸운다.

## 4. 핵심 메카닉

### 4.1 퍼즐 보드

- **형태:** 헥사고날 그리드 5열 x 6행 (검증 후 수정 가능)
- **젬 종류:** 5가지 — 불(Fire), 물(Water), 풀(Grass), 빛(Light), 어둠(Dark)
- **젬 스폰 확률:** 기본 각 20%, 파티 구성에 따라 보정
  - 해당 속성 캐릭터 1명당 +5~10% 보정, 초과분은 나머지 속성에서 균등 차감

### 4.2 드래그 시스템

- 플레이어가 젬을 터치/클릭 후 인접 젬으로 드래그하여 연결
- 같은 젬 재방문 가능 여부는 추후 검증
- 드래그 중 연결 경로를 색상별 선으로 실시간 시각화
- **드래그 종료 시 무조건 현재 캐릭터 교체** (스킬 발동 여부 무관)

### 4.3 시퀀스 시스템

- 각 캐릭터는 고유한 스킬 시퀀스 보유 (예: 불→불→물, 빛→어둠→빛→풀)
- 드래그로 연결한 젬 순서가 현재 활성 캐릭터의 시퀀스와 일치하면 스킬 발동
- 불일치 시 기본 공격 또는 효과 없음
- 보드 하단에 현재 활성 캐릭터의 시퀀스 항상 표시

### 4.4 캐릭터 순환

- 아군 4명이 A→B→C→D→A 순서로 순환
- 드래그 종료마다 다음 캐릭터로 교체 (스킬 발동 여부 무관)
- 현재 활성 캐릭터는 시각적으로 강조 (앞으로 나오거나 빛남)

### 4.5 적 실시간 공격

- 각 적은 개별 공격 타이머 보유
- 타이머가 차면 즉시 아군에게 공격 (별도 적 턴 없음)
- 적 공격 게이지를 UI에 실시간 표시

## 6. 스테이지 구조

- 각 스테이지는 적 구성(종류, 수, HP, 공격력, 공격속도)을 데이터로 정의
- 클리어 조건: 모든 적 처치
- 실패 조건: 아군 전원 HP 0
- MVP: 스테이지 3~5개, 이후 데이터 추가만으로 확장

---

## 7. MVP 범위

이번 구현에서 포함:
- 헥사 퍼즐 보드 렌더링 + 드래그 연결
- 5속성 젬 + 파티 기반 스폰 확률
- 4인 파티 순환 + 시퀀스 검증 + 스킬 발동
- 적 실시간 공격 타이머
- 스테이지 3~5개 클리어 구조
- 기본 UI (HP바, 시퀀스 표시, 공격 게이지)

이번 구현에서 제외 (추후):
- 캐릭터 성장/수집 시스템
- 장비, 스킬트리
- 모바일/Steam 래핑
- 사운드, 이펙트 고도화

## 8. 데이터 구조 예시

### 캐릭터
```js
{
  id: 'warrior',
  name: '전사',
  element: 'fire',
  hp: 1000,
  attack: 150,
  skillSequence: ['fire', 'fire', 'water'],
  skillName: '불꽃 베기',
  skillEffect: { type: 'damage', multiplier: 3.0 }
}
```

### 적
```js
{
  id: 'goblin',
  name: '고블린',
  hp: 500,
  attack: 80,
  attackInterval: 4000  // ms
}
```

### 스테이지
```js
{
  id: 1,
  name: '첫 번째 시련',
  enemies: [
    { id: 'goblin', count: 3 }
  ]
}
```
